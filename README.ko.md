# dd

> 긴 로그나 스크린샷을 대화창에 통째로 붙이지 않고 Claude Code에 넘기는 방법.

Claude Code로 작업할 때 우리는 붙여넣기를 정말 많이 합니다. 디버깅할 땐 에러 로그를 던지고, 뭘 만들 땐 레퍼런스를 보여주고, 원하는 화면을 캡처해서 설명하죠. `dd`는 그걸 클립보드에서 바로 읽어 로컬 파일에 두고, 대화엔 짧은 요약(민감정보는 가림)만 넘깁니다.

## 빠른 시작

### 1. 마켓플레이스 추가

```
/plugin marketplace add https://github.com/fivetaku/gptaku_plugins.git
```

### 2. 설치

```
/plugin install dd
```

### 3. Claude Code 재시작

(새 명령은 재시작해야 등록됩니다.)

### 4. 사용법

클립보드에 뭔가 올린 뒤, `/dd`(또는 `/ㅇㅇ`)에 요청을 붙여 입력하면 됩니다 — **붙여넣기 안 해도 돼요**:

- **이미지 — Windows:** `Win`+`Shift`+`S`로 캡처 → `/dd 왜 이렇게 보여?` (캡처가 이미 클립보드에 있음)
- **이미지 — macOS:** `Shift`+`Cmd`+`4`로 캡처 → `/dd 이 레퍼런스처럼 만들어줘` (처음 한 번만, 스크린샷이 클립보드로 저장되게 설정: 도구막대 `Shift`+`Cmd`+`5` → **옵션** → **저장 위치: 클립보드**)
- **긴 텍스트 / 에러 로그:** 복사 → `/dd 무슨 에러야?` — 대화창에 긴 텍스트가 통째로 안 들어가서 세션이 가볍게 유지됨
- **요청 없이:** 그냥 `/dd` — 클립보드를 읽고 대화 맥락을 이어 처리

한글 입력 상태에선 `/dd`를 그대로 쳐도 `/ㅇㅇ`로 동작해요. 한영 전환 불필요.

## 왜 dd인가

Claude Code로 작업하면 붙여넣을 일이 끝없이 생깁니다. 디버깅할 땐 에러 로그를 던지고, 뭘 만들 땐 레퍼런스를 보여주고, 원하는 화면을 캡처해서 "이런 느낌" 하고 설명하죠. 그런데 이걸 대화창에 직접 붙이면 두 가지가 걸립니다.

긴 텍스트는 대화를 잡아먹습니다. 긴 로그를 한 번 붙이면 계속 남아 자리를 차지하고, 턴마다 다시 읽혀서 점점 느리고 비싸지죠. `dd`는 원문을 파일에 두고 짧은 요약만 넘긴 다음, 필요한 만큼만 읽습니다.

이미지는 아예 붙여넣기가 안 되기도 합니다. 환경에 따라 Claude Code가 붙여넣은 이미지를 못 받거든요. `dd`는 클립보드를 직접 읽으니 캡처하고 `/dd`만 치면 됩니다. 복사한 이미지 파일도 아이콘이 아니라 실제 그림으로 가져오고요.

덤으로 "방금 복사한 에러"나 "위 레퍼런스" 같은 설명을 반복할 필요가 없고, 엉뚱한 걸 잡으면(클립보드엔 복사 시각이 없으니) 뭘 잡았는지 보여주고 먼저 물어봅니다.

이름이 `dd`인 건 별 뜻 없어요. 자주 쓰는데 매번 길게 치긴 귀찮으니 빨리 두 번 두드리는 두 글자로 정한 겁니다. 한글 상태에선 `dd`가 `ㅇㅇ`으로 찍히는데, 그것도 똑같이 동작합니다.

## 작동 방식

```
복사 / 스크린샷
   → /dd [요청]
   → dd_clipboard.py가 현재 클립보드를 ~/dd/<날짜>/<id>/ 에 캡처
       · 텍스트 → content.txt   이미지 → image.png   + manifest.json
   → Claude는 manifest 요약만 읽음 (원문 전체 X)
   → 뭘 잡았는지 보여주고, 이상하면 확인, 맞으면 바로 작업
```

OS 클립보드는 가장 최근 복사 1개만 담습니다(이력 없음). `dd`는 항상 그 현재 항목을 잡고, preview에서 민감정보를 가리며, 7일 지난 캡처는 자동 정리합니다.

## 주요 기능

| 기능 | 설명 |
|------|------|
| 텍스트·이미지 캡처 | macOS / Windows / WSL / Linux 클립보드, 텍스트 또는 이미지 |
| instruction 패턴 주입 | 매 `/dd`마다 `${CLAUDE_PLUGIN_ROOT}`로 캡처 — 누구나 동작 |
| 확인 게이트 | 클립보드가 요청과 무관하거나 애매할 때만 작업 전 확인 |
| 지연 읽기 | `size_class`별로 읽어 거대한 붙여넣기가 컨텍스트를 안 덮침 |
| 민감정보 마스킹 | `api_key`, `Bearer`, `sk-`, `ghp_`, `xoxb-` 등 preview에서 가림 |
| 자동 정리 | `DD_RETENTION_DAYS`(기본 7일) 지난 캡처를 실행 시 삭제 |

| 명령어 | 설명 |
|--------|------|
| `/dd [요청]` | 현재 클립보드를 캡처해 작업 |
| `/ㅇㅇ [요청]` | `/dd`와 동일 (한글 입력 별칭) |

### 자연어 트리거

- "방금 캡처한 거 분석해줘", "이 레퍼런스로 만들어줘", "스크린샷 드롭"
- "drop clipboard", "use what I copied", "this screenshot"

## 요구사항

- [Claude Code](https://docs.anthropic.com/claude-code) CLI
- Python 3 (표준 라이브러리만)
- **macOS**: 바로 동작 (`pbpaste` / `osascript`)
- **Windows / WSL**: PowerShell(기본 내장) — 사용자 검증
- **Linux**: 클립보드 도구 설치(`xclip` / `wl-clipboard`) + GUI 세션 필요
